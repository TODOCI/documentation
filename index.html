Todofy Documentation

Simple, flexible and extensive todo manager for your software development projects, Crafted with ♥ by todofy

Sign up using Github

By this time there are around ~ 89 million TODO, ~ 45 million FIXME and several other HACK, TBD etc in Github repositories. Using todofy one can keep track of their todos, and use other features to make sure they finish those.

Signup Simply login using Github Account

It's very very easy. You can sign in / sign up any time using Github. Click here to sign in / sign up now!
Todofy requires your permission to read your email id, read/write access to all your public repositories and read-access to your organizations and teams. This is needed to check collaborator access to repositories and add webhooks to track status of all todos after each commit. We do not need write permission to your public repositories, but it comes bundled in. It's all about trust :)

Repositories Integrate all your public repositories and organisation reporitories!

When you sign up, we list out each of your public repositories and organizations repositories. Also, you can refresh this list anytime from your profile page.
You can refresh using the Sync button on top right.


Organizational repositories

You can visit any organization page by visiting their url https://todofy.org/r/<org name>.
For example: https://todofy.org/r/todofy.

Or if you are a team member / collaborator at any organization, you can view them on your profile page by clickin on the listed organizations.

Todofication Todofy a repository once, and we will sync every commits you push!

You can todofy a repository from your profile page, or by visiting the repository page at: https://todofy.org/r/<repo full name>.

Note that you can todofy multiple repositories at any time but they are queued up and done one by one. Also, note that todofication time depend on size of repository and no of branches. Once the repository is todofied, the summary is sent to all the collaborators of the project. After todofication of the master branch is finished it will start listing out all active todos:

Labels Add labels to group your todos to categories!

Labels are used to group similar todos into categories and thus for filtering todos in the list. Here's information how you can add it both using the dashboard and directly while adding todo.

From the dashboard:

You can also add labels without modifying the comments directly from the dashboard. Go to the todo page on Todofy and in the right hand menu, click on labels and select an existing one or add a new one.

Directly in code comment:

You can add labels by adding @label parameter in your comments. For example in C++: 
int getSum(int a, int b) {
  // TODO: make this a template function @label: template, modification
  return a + b;
}
This will add labels template & modification to this todo.

Tags Tag other users to pull them to disucssion!

You can tag other users in a todo to notify them and bring them to discussion. As soon as a tag is added, Todofy notifies respective users about the todo.

From the dashboard:

You can also tag users without modifying the todo comments directly from the dashboard. Go to the todo page and in the right hand menu, click on Tags and select an existing one or add a new one by entering their Github username.

Directly in code comment:

You can Tag users with their Github username using @tag parameter in your comments. For example in C++: 
int getSum(int a, int b) {
  // TODO: make this a template function @tag: mebjas, abhinavdahiya
  return a + b;
}
This will Tag user mebjas & abhinavdahiya and notify them.

Deadline Set deadline on todos, to finish them on time

You can set a deadline for a todo, Todofy will track its progress and notify you if a todo crosses it's deadline. Also you can see all your upcoming deadlines (for todos from different repositories) in the dashboard. Similarly you can check all your expired todos from the same page. You can set deadline both from dashboard and code.

From the dashboard:

You can set deadline for a todo from the right hand side menu, click on set deadline > select a date > click on set.

Directly in code comment:

You can add a deadline using @deadline parameter in your comments. For example in C++: 
int getSum(int a, int b) {
  // TODO: make this a template function @deadline: 1 week
  return a + b;
}
This will set a deadline of 1 week for the todo. In a similar way you can set it as:

now

10 September 2016

+1 day

+1 week

+1 week 2 days 4 hours 2 seconds

next Thursday

last Monday

yyyy-mm-dd

yy-mm-dd, note in this case if yy is between 00-69 it will be mapped as 2000-2069, else it will be mapped as 1970-1999.

Reminder Set reminder on todos, to make sure you finish them on time

You can set reminder for a todo, Todofy will notify you when it's time. You can set reminder both from dashboard and code.

From the dashboard:

You can set reminder for a todo from the right hand side menu, click on set reminder > select a date > click on set.

Directly in code comment:

You can add a reminder using @reminder parameter in your comments. For example in C++: 
int getSum(int a, int b) {
  // TODO: make this a template function @reminder: 3 day
  return a + b;
}
This will set a deadline of 1 week for the todo. In a similar way you can set it as:

now

10 September 2016

+1 day

+1 week

+1 week 2 days 4 hours 2 seconds

next Thursday

last Monday

yyyy-mm-dd

yy-mm-dd, note in this case if yy is between 00-69 it will be mapped as 2000-2069, else it will be mapped as 1970-1999.

Priority Set priority on todos, to organise them!

You can set priority for a todo to organise them in the list. It can be added both from dashboard and from the todo comment.

From the dashboard:

You can set priority for a todo from the right hand side menu, click on set priority > select a priority

Directly in code comment:

You can set priority using @priority parameter in your comments. For example in C++: 
int getSum(int a, int b) {
  // TODO: make this a template function @priority: max
  return a + b;
}
Note that by default High, Medium, Low priorities are defined in descending order. If a new priority is added from code, its added to the system and given lowest priority. You can change the order of priority from the Repository Settings. Also priority is uniform for all the branches in a repository.

Create Github Issue Add a todo as a Github issue

You can create a Github issue for any todo. Currently it's possible only through dashboard. But we are working on adding a parameter to auto create a Github issue.
You can add a Github issue for a todo by clicking on CREATE GITHUB ISSUE button on right hand side.
Then fill in the description about this issue.
Note the issue will be created by our bot user fybot. We do not have permissions to auto create a issue in a public repository with your credentials.

Discussion Discuss about todos directly using todofy!

The comment feature can be used by users to discuss about a todo. Currently it doesn't support HTML or markdown. We are working on adding Markdown support and options to mention users or cite links.

To Read Add articles you need to read / look at to finish a todo!

Collaborators of a repository can add links in to read section, available on right hand side menu. Note: this feature is available only through dashboard. To add a link to this list, click on add & enter the link and Save. If the link is avialble, it will be added. It can be mark as read by clicking on the checkbox.
